## GitHub Widgets

The statistics displayed above are generated by a self-hosted Vercel API at [`bkataru-widgets.vercel.app`](https://bkataru-widgets.vercel.app). The source code is available in the [`vercel-api/`](./vercel-api) directory.

### Data Sources

The widgets aggregate data from:
- My personal GitHub account (`bkataru`)
- 17 organizations I'm part of, including: `bkataru-workshop`, `bkataru-experiments`, `bkataru-recreations`, `bkataru-playgrounds`, `bkataru-forks`, `bkataru-vaults`, and others

### Language Statistics Methodology

GitHub's default language statistics are calculated purely by **bytes of source code**, which can be misleading. For example, a single repository with auto-generated Mathematica notebook outputs (89MB) would dominate the chart despite representing minimal actual coding effort.

To provide a more accurate representation of language usage, this project uses a **normalized scoring algorithm**:

```
score = sqrt(bytes) √ó log(repos + 1)
```

**Why this formula works:**

| Component | Purpose |
|-----------|---------|
| `sqrt(bytes)` | Dampens the impact of extremely large files. A 90MB file becomes ~9,500 instead of 90,000,000 |
| `log(repos + 1)` | Rewards languages used across multiple repositories, indicating broader usage patterns |

**Example Impact:**

| Language | Raw Bytes % | Normalized % | Repos |
|----------|-------------|--------------|-------|
| Mathematica | 58.2% | 14.7% | 3 |
| Python | 14.8% | **23.1%** | 75 |
| Rust | 2.6% | **9.1%** | 58 |
| TypeScript | 2.2% | **7.0%** | 29 |

This approach better reflects actual coding activity by balancing code volume with breadth of usage.
